# Project 4 — 조도센서 (Light Sensor Module)

4핀 조도센서 모듈을 사용합니다.  
빛이 밝으면 값이 낮고, 어두우면 높습니다 — 직관적으로 "어둠의 정도"를 측정한다고 생각하면 됩니다.

---

## 배선

```
조도센서 모듈 (4핀):
  VCC → 5V
  GND → GND
  AO  → A0  (아날로그 출력)
  DO  → 미사용 (디지털 출력, 이 프로젝트에서는 안 씀)
```

---

## 프로토콜

```
[주소: 6] [값: 0~255]
```

0 = 밝음, 255 = 어두움.  
50ms 간격 전송.

아두이노에서 `map(raw, 0, 1024, 255, 0)`으로 값을 뒤집습니다.  
센서 원본은 밝을 때 높은 값이 나오는데, 직관적으로 "어두움=높은값"이 되도록 반전시킨 겁니다.

---

## 01. Basic — 기본 통신

**목표:** 밝기값 실시간 확인

Max 패치 구조는 P1, P2와 거의 동일합니다:
- `serial` → `zl group 2` → `route 6` → `number` → `print`

손으로 센서를 가리면 값이 올라가고, 손전등을 비추면 내려갑니다.

---

## 02. Application — 이미지 밝기 조절

**목표:** 센서가 어두우면 화면의 이미지도 어두워짐

새로 등장하는 Max 오브젝트:
- `jit.matrix 4 char 320 240` — 이미지를 담는 매트릭스
- `jit.brcosa` — Brightness(밝기), Contrast(대비), Saturation(채도) 조절
- `brightness $1` 메시지로 밝기를 0.0~1.0 범위로 제어

이미지 로딩:
- `loadbang` → `importmovie brick_colormap.jpg` — Max 내장 이미지를 자동으로 불러옴
- `qmetro 33`으로 Display 루프를 돌려야 화면에 보입니다

센서 → `scale 0 255 0. 1.` → `brightness $1` → `jit.brcosa`

방을 어둡게 하면 화면도 같이 어두워지는 간단하지만 효과적인 매핑입니다.

---

## 03. MediaArt — 영상 어둡게 + 소리 먹먹하게

**목표:** 어둠에 반응하는 멀티미디어 설치 — 영상과 오디오를 동시에 제어

하나의 센서값(0~255)에서 4개의 파라미터를 뽑아냅니다:

| 매핑 | scale 범위 | 효과 |
|------|-----------|------|
| 밝기 | 0~255 → 0.0~1.0 | `jit.brcosa` brightness |
| 볼륨 | 0~255 → 0.0~1.0 | `*~`로 음량 조절 |
| 필터 | 0~255 → 150~8000Hz | `lores~` 로우패스 컷오프 |
| 리버브 | 0~255 → 500~10ms | `tapout~` 딜레이 타임 |

밝으면(값이 낮으면):
- 영상 밝고 선명
- 소리 크고 또렷

어두우면(값이 높으면):
- 영상 어둡게
- 소리 작고 먹먹하게 + 울림 많아짐

오디오 체인:
```
jit.movie~ (영상+오디오 동시 출력)
    ↓ (signal)
lores~ ← line~ (필터 주파수)
    ↓
*~ ← line~ (볼륨)
    ↓
+~ (드라이) ←── *~ 0.4 ←── tapout~ ←── tapin~
    ↓                                      ↑
gain~ → ezdac~              (리버브 피드백 루프)
```

`jit.movie~`를 쓰는 게 포인트입니다. `jit.movie`가 아니라 `jit.movie~` — 물결표가 붙으면 오디오 시그널도 같이 출력됩니다. 그래서 영상의 사운드트랙을 필터링할 수 있는 거예요.

**사용 순서:**
1. Open Port → Start → Render
2. ezdac~ 켜기
3. `read crashtest.mov` 클릭 + `loop 1` 클릭
4. gain~ 슬라이더 올리기
5. 센서를 손으로 가리거나 불을 끄면 효과 확인
