# Project 3 — 서보모터 (SG90)

여기서부터 데이터 방향이 바뀝니다.  
Project 1~2는 "아두이노 → Max"였는데, 서보모터는 **"Max → 아두이노"**입니다.  
Max가 계산한 값을 시리얼로 보내면 아두이노가 모터를 움직입니다.

---

## 배선

```
서보모터 SG90:
  주황 (Signal) → 디지털 9번
  빨강 (VCC)    → 5V
  갈색 (GND)    → GND
```

---

## 프로토콜

Max에서 아두이노로 **1바이트** (0~180)를 보내면 서보가 해당 각도로 이동합니다.  
주소 바이트 없이 값 하나만 보내는 단순한 구조.

---

## 01. Basic — 기본 통신

**목표:** Max의 number 박스에 숫자를 넣으면 서보가 해당 각도로 움직임

아두이노 코드:
- `Serial.read()`로 1바이트 읽기
- `constrain(angle, 0, 180)`으로 범위 제한
- `myServo.write(angle)`로 적용

Max 패치는 매우 단순합니다. number 박스 → serial로 직접 연결.  
number에 0을 넣으면 왼쪽 끝, 90이면 정면, 180이면 오른쪽 끝.

---

## 02. Application — 서보 메트로놈

**목표:** BPM에 맞춰 서보가 좌우로 왕복 + 틱 사운드

동작 원리:
1. BPM과 비트 배수(1×, 2×, 4×)를 설정
2. `expr 60000./($f1*$i2)` — BPM과 배수로 비트 간격(ms)을 계산
3. `metro`가 해당 간격으로 bang을 발생
4. `counter 0 1` — 0과 1을 번갈아 출력
5. 0이면 서보 45도(왼쪽), 1이면 135도(오른쪽)

사운드:
- `click~` → `reson~ 1000 800 0.8` — 메트로놈 틱 소리를 만듦
- bang이 올 때마다 짧은 임펄스가 bandpass 필터를 통과해서 "똑" 소리가 남

**비트 배수 설정:**
- 1 = 4분음표 (기본)
- 2 = 8분음표
- 4 = 16분음표

메시지 박스를 더블클릭해서 선택합니다.

---

## 03. MediaArt — 소리 반응 서보

**목표:** 마이크로 소리를 감지해서 소리 크기에 따라 서보가 움직임

방향이 좀 독특합니다:
```
마이크 → Max(peakamp~로 음량 측정) → 시리얼 → 아두이노 → 서보
```

Max 쪽 처리:
1. `adc~` — 마이크 입력
2. `qmetro 50` + `peakamp~` — 50ms마다 피크 진폭 측정 (0.0~1.0)
3. `scale 0. 0.5 0 180` — 진폭을 각도로 변환 (0.5 이상은 180도로 클램프)
4. `int` — 정수로 변환 후 시리얼로 전송

조용하면 서보가 0도(왼쪽 끝), 큰 소리를 내면 180도(오른쪽 끝)로 확 움직입니다.  
박수를 치거나 음악을 틀면 서보가 반응하는 걸 볼 수 있습니다.

**주의:** ezdac~을 먼저 켜야 마이크가 활성화됩니다. Open Port → ezdac~ → Start 순서.
